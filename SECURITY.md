# Security Policy

SceauID is currently pre-1.0 and should be treated as experimental identity infrastructure.

## Supported Versions

No production-stable version has been released yet.

## Reporting a Vulnerability

Please do not open a public issue for sensitive security findings.

Report suspected vulnerabilities by emailing:

```txt
kehindeibukun0619@gmail.com
```

Include:

- affected component
- reproduction steps
- expected impact
- suggested mitigation, if known

## Security Scope

Security-sensitive areas include:

- passkey registration and authentication
- session creation, validation, and revocation
- recovery codes and recovery requests
- rate limiting and abuse controls
- security event integrity
- SDK and framework integration boundaries

## Current Status

SceauID is not production-ready. Security documentation and threat modeling will evolve alongside implementation.
